# LiveKit Module

LiveKit powers voice and video rooms. Without it, voice channels are disabled but the rest of the app works normally.

## Variables the wizard needs from the user

| Variable              | Description                             | Example                |
| --------------------- | --------------------------------------- | ---------------------- |
| `LIVEKIT_SERVER_IP`   | Public IP of the server running LiveKit | `89.548.32.568`        |
| `LIVEKIT_DOMAIN`      | Domain for LiveKit WebSocket            | `media.yourdomain.com` |
| `LIVEKIT_TURN_DOMAIN` | Domain for TURN server                  | `turn.yourdomain.com`  |

`LIVEKIT_API_KEY` and `LIVEKIT_API_SECRET` are generated automatically by the wizard.

## Files generated by the wizard

The wizard produces the following files on the LiveKit server:

- `livekit.yaml` — LiveKit config (generated from `livekit.example.yaml`)
- `caddy.yaml` — Caddy L4 SNI routing config (generated from `caddy.example.yaml`)
- `docker-compose.yaml` — compose with livekit, caddy, and redis
- `redis.conf` — local Redis config

## Variables to copy into VPS1 .env

```
LIVEKIT_API_KEY=<generated>
LIVEKIT_API_SECRET=<generated>
NEXT_PUBLIC_LIVEKIT_URL=wss://media.yourdomain.com
```

## Required firewall ports on the LiveKit server

- `443/tcp` — Caddy (LiveKit WebSocket + TURN TLS)
- `3478/udp` — TURN UDP
- `50000-60000/udp` — RTC media (range configurable in `livekit.yaml`)
- `7881/tcp` — RTC TCP fallback
